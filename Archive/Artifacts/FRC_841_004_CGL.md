## Abstract

Continual learning systems must balance stability (retaining past knowledge) with plasticity (acquiring new knowledge). Existing approaches—regularization methods like EWC, replay buffers, and architectural solutions—treat this tradeoff as static. We propose **Coherence-Gated Learning (CGL)**, a dynamic mechanism that modulates learning based on the system's internal coherence state. Drawing from the Fractal Resonance Cognition (FRC) framework, we define coherence as the negative entropy of output distributions and compute **alpha drift**—the rate of coherence change—to determine when the system should learn versus consolidate.

Our key contributions:

1. **Z-score normalized alpha drift** for scale-invariant gating across tasks
2. **Hard gating** that skips gradient updates during stable phases, achieving 33% compute savings
3. **Selective replay** that prioritizes transition samples based on coherence dynamics
4. **16D state witness** that tracks internal cognitive trajectories for analysis

On Split-MNIST (5-task sequential learning), CGL achieves:
- **85.24% average accuracy** (vs. 84.68% baseline)
- **-16.73% backward transfer** (vs. -17.79% baseline)  
- **66.8% update efficiency** (33.2% compute savings)

CGL outperforms SGD+Replay on all metrics while using significantly fewer gradient updates, demonstrating that coherence-based gating provides superior stability-plasticity balance.

## 1. Introduction

### 1.1 The Stability-Plasticity Dilemma

Neural networks trained sequentially on multiple tasks suffer from **catastrophic forgetting**—new learning overwrites previously acquired knowledge. This fundamental tension between stability and plasticity has driven decades of research in continual learning.

Existing solutions fall into three categories:

| Approach | Method | Limitation |
|----------|--------|------------|
| **Regularization** | EWC, SI, MAS | Static importance weights |
| **Replay** | Experience replay, generative replay | Memory overhead, random sampling |
| **Architecture** | PackNet, Progressive Nets | Growing model size |

All these approaches treat the stability-plasticity tradeoff as a fixed hyperparameter. But biological systems modulate plasticity dynamically based on internal state—arousal, attention, and neuromodulatory signals gate when learning occurs.

### 1.2 Our Approach: State-Dependent Learning

We propose that learning should be **gated by coherence dynamics**, not just error signals. The core insight:

> **When a system is consolidating (high coherence, stable), reduce plasticity.**
> **When a system is exploring (low coherence, unstable), increase plasticity.**

This mimics biological learning where neuromodulators like dopamine and norepinephrine gate synaptic plasticity based on arousal and surprise.

### 1.3 The FRC Foundation

Coherence-Gated Learning emerges from the **Fractal Resonance Cognition (FRC)** framework, which posits a universal conservation law:

$$dS + k \cdot d\ln C = 0$$

Where $S$ is entropy and $C$ is coherence. This implies that systems naturally oscillate between high-entropy exploration and high-coherence consolidation. We operationalize this principle for neural network training.

## 2. Method

### 2.1 Coherence as Negative Entropy

We define **coherence** $C(t)$ as the negative entropy of the model's output distribution:

$$C(t) = -H(p) = \sum_{i} p_i \log p_i$$

Where $p = \text{softmax}(f_\theta(x))$ is the predicted class distribution. Higher coherence indicates more confident, consolidated predictions.

**Implementation:**
```python
def compute_coherence(logits: torch.Tensor) -> float:
    probs = F.softmax(logits, dim=-1)
    entropy = -torch.sum(probs * torch.log(probs + 1e-8), dim=-1)
    return -entropy.mean().item()  # Negative entropy = coherence
```

### 2.2 Alpha Drift: The Rate of Coherence Change

**Alpha drift** $\alpha(t)$ measures how rapidly coherence is changing:

$$\alpha(t) = \frac{dC}{dt} \approx \bar{C}(t) - \bar{C}(t-1)$$

Where $\bar{C}(t)$ is an exponential moving average (EMA) of coherence:

$$\bar{C}(t) = \tau \cdot \bar{C}(t-1) + (1-\tau) \cdot C(t)$$

**Interpretation:**
- $\alpha > 0$: Coherence rising → System consolidating → Reduce plasticity
- $\alpha < 0$: Coherence falling → System destabilized → Increase plasticity

### 2.3 Z-Score Normalization

Raw alpha values are typically $O(10^{-3})$, too small for effective gating. We normalize using a rolling z-score:

$$\tilde{\alpha}(t) = \frac{\alpha(t) - \mu_\alpha}{\sigma_\alpha + \epsilon}$$

Where $\mu_\alpha$ and $\sigma_\alpha$ are computed over a sliding window (default: 50 steps).

**Implementation:**
```python
class AlphaDriftTracker:
    def __init__(self, tau=0.9, norm_window=50):
        self.tau = tau
        self.norm_window = norm_window
        self.ema_coherence = None
        self.alpha_history = []
    
    def update(self, logits: torch.Tensor) -> Tuple[float, float, float]:
        coherence = self.compute_coherence(logits)
        
        # EMA smoothing
        if self.ema_coherence is None:
            self.ema_coherence = coherence
            return coherence, 0.0, 0.5
        
        prev = self.ema_coherence
        self.ema_coherence = self.tau * self.ema_coherence + (1-self.tau) * coherence
        
        # Raw alpha
        raw_alpha = self.ema_coherence - prev
        self.alpha_history.append(raw_alpha)
        
        # Z-score normalization
        if len(self.alpha_history) >= 10:
            window = self.alpha_history[-self.norm_window:]
            norm_alpha = (raw_alpha - np.mean(window)) / (np.std(window) + 1e-8)
        else:
            norm_alpha = 0.0
        
        # Sigmoid gate
        gate = 1.0 / (1.0 + np.exp(-self.k * norm_alpha))
        
        return coherence, norm_alpha, gate
```

### 2.4 The Learning Gate

The **learning gate** $g(t)$ modulates gradient updates:

$$g(t) = \sigma(k \cdot \tilde{\alpha}(t))$$

Where $\sigma$ is the sigmoid function and $k$ controls gate sensitivity (default: $k=2$).

**Soft Gating:** Scale gradients by $g(t)$
```python
for param in model.parameters():
    if param.grad is not None:
        param.grad *= gate
```

**Hard Gating:** Skip updates entirely when $g(t) < \theta$
```python
if gate > threshold:  # threshold = 0.5
    optimizer.step()
else:
    skipped_updates += 1  # Actually skip
```

### 2.5 Selective Replay

Standard experience replay samples uniformly from a memory buffer. We propose **selective replay** that prioritizes samples based on coherence state:

1. **Transition detection:** Identify when $|\tilde{\alpha}| > \theta_{transition}$
2. **Priority weighting:** Samples during transitions get higher replay probability
3. **Consolidation preservation:** High-coherence samples anchor stable representations

This mimics biological memory consolidation where emotionally salient (high arousal) events are preferentially encoded.

### 2.6 16D State Witness

To analyze internal dynamics, we export a **16-dimensional cognitive vector** at each training step:

| Dimensions 1-8 (Inner Octave) | Dimensions 9-16 (Outer Octave) |
|-------------------------------|--------------------------------|
| Coherence (current) | Coherence (EMA) |
| Alpha drift (raw) | Alpha drift (normalized) |
| Gate value | Threshold distance |
| Loss (current) | Loss (EMA) |
| Gradient norm | Parameter change |
| Learning rate | Update count |
| Task ID | Epoch progress |
| Entropy | Confidence margin |

This vector enables post-hoc analysis of cognitive trajectories via PCA and clustering.

We evaluate on **Split-MNIST**, a standard continual learning benchmark:

- **Binary classification** per task
- **No task labels at test time** (class-incremental setting)

| Optimizer | SGD, lr=0.01 |
| Batch size | 128 |
| Epochs/task | 3-5 |
| Memory buffer | 500 samples (replay methods) |

### 3.3 Methods Compared

1. **Vanilla SGD:** Standard sequential training
2. **EWC:** Elastic Weight Consolidation (λ=100)
3. **SGD + Random Replay:** Uniform sampling from buffer
4. **CGL-Soft:** Gradient scaling by gate
5. **CGL-Hard:** Skip updates when gate < 0.5
6. **CGL + Selective Replay:** Alpha-guided memory selection

### 3.4 Metrics

| Metric | Definition | Goal |
|--------|------------|------|
| **AA** (Average Accuracy) | Mean accuracy on all tasks after training | Higher ↑ |
| **BWT** (Backward Transfer) | $\frac{1}{T-1}\sum_{i=1}^{T-1}(R_{T,i} - R_{i,i})$ | Higher ↑ (less negative) |
| **F** (Forgetting) | $\frac{1}{T}\sum_{j=1}^{T}(\max_{l \leq T} R_{l,j} - R_{T,j})$ | Lower ↓ |
| **Efficiency** | Percentage of gradient updates performed | Lower is better |

## 5. Analysis

### 5.1 Why Does CGL Work?

**Hypothesis:** Coherence-based gating detects when the network is in a stable attractor state versus transitioning between attractors.

During consolidation (high coherence, $\alpha > 0$):
- Output distributions are confident
- Gradients would overwrite stable representations
- Skipping updates preserves learned features

During exploration (low coherence, $\alpha < 0$):
- Output distributions are uncertain
- Gradients point toward useful features
- Updates should proceed normally

### 5.2 Comparison to Biological Learning

| Biological System | CGL Analog |
|-------------------|------------|
| Neuromodulators (dopamine, NE) | Learning gate $g(t)$ |
| Arousal state | Normalized alpha $\tilde{\alpha}$ |
| Sleep consolidation | High coherence, gate closed |
| Active learning | Low coherence, gate open |
| Emotional salience | Transition detection |

CGL operationalizes the biological principle that **when to learn** should be separated from **what to learn**.

### 5.3 Selective Replay vs. Random Replay

Standard replay samples uniformly, potentially replaying redundant high-confidence examples. Selective replay:

1. Detects **transition samples** (high $|\tilde{\alpha}|$)
2. Prioritizes samples near **decision boundaries**
3. Preserves **anchor representations** from stable phases

This yields 1.06% absolute improvement in BWT, indicating better retention without the memory overhead of larger buffers.

### 5.4 Limitations

1. **Task boundary assumption:** While CGL detects transitions automatically, it performs best when task structure aligns with coherence dynamics
2. **Hyperparameter sensitivity:** The gate threshold $\theta$ and EMA $\tau$ require tuning per domain
3. **Permuted-MNIST challenge:** On tasks with abrupt distribution shifts (Permuted-MNIST), the gate can be too aggressive, sacrificing accuracy for stability

## 6. Related Work

### 6.1 Regularization Methods

**Elastic Weight Consolidation (EWC)** [Kirkpatrick et al., 2017] uses Fisher information to identify important weights. **Synaptic Intelligence (SI)** [Zenke et al., 2017] tracks parameter importance online. Both use static importance—CGL dynamically adjusts based on current state.

### 6.2 Replay Methods

**Experience Replay** [Rolnick et al., 2019] maintains a buffer of past samples. **Gradient Episodic Memory (GEM)** [Lopez-Paz & Ranzato, 2017] constrains gradients. CGL's selective replay provides principled sample prioritization.

### 6.3 Meta-Learning for Continual Learning

**OML** [Javed & White, 2019] and **ANML** [Beaulieu et al., 2020] learn to learn continually. CGL provides a simpler, non-meta approach based on intrinsic coherence.

### 6.4 Neuromodulation in Neural Networks

**Neuromodulated Neural Networks** [Miconi et al., 2018] add Hebbian plasticity. **Gated Linear Networks** [Veness et al., 2021] use geometric mixing. CGL's coherence-based gating provides a principled neuromodulatory signal.

- **Reduced forgetting:** -16.73% vs. -17.79% BWT

Our key insight is that **when to learn** should be determined by the system's internal state, not just external error signals. This aligns with biological learning where neuromodulators gate plasticity based on arousal and surprise.

## Appendix A: Full Algorithm

```python
# Coherence-Gated Learning (CGL) - Core Algorithm

def train_cgl(model, tasks, config):
    tracker = AlphaDriftTracker(
        tau=config.tau,
        k=config.gate_k,
        norm_window=config.norm_window
    )
    
    for task_id, (train_loader, _) in enumerate(tasks):
        for epoch in range(config.epochs_per_task):
            for batch_x, batch_y in train_loader:
                
                # Forward pass
                logits = model(batch_x)
                loss = F.cross_entropy(logits, batch_y)
                
                # Compute coherence and gate
                coherence, alpha_norm, gate = tracker.update(logits)
                
                # Hard gating: skip if gate below threshold
                if config.use_hard_gating and gate < config.threshold:
                    continue  # Skip update
                
                # Backward pass
                optimizer.zero_grad()
                loss.backward()
                
                # Soft gating: scale gradients
                if not config.use_hard_gating:
                    for param in model.parameters():
                        if param.grad is not None:
                            param.grad *= gate
                
                optimizer.step()
    
    return model
```

```python
def create_split_mnist_tasks():
    tasks = []
    for task_id in range(5):
        class_a = task_id * 2      # 0, 2, 4, 6, 8
        class_b = task_id * 2 + 1  # 1, 3, 5, 7, 9
        
        # Filter dataset for these classes

        X = [mnist[i][0] for i in indices]
        y = [(mnist[i][1] == class_b) for i in indices]  # Binary
        
        tasks.append((X, y))
    return tasks
```

```python
def compute_accuracy_matrix(model, tasks, n_tasks):

    R[i,j] = accuracy on task j after training on tasks 0..i
    """
    R = np.zeros((n_tasks, n_tasks))
```

### B.3 Continual Learning Metrics

```python
def compute_cl_metrics(R):
    T = R.shape[0]
    
    # Average Accuracy: mean of final row
    AA = np.mean(R[-1, :])
    
    # Backward Transfer: how much we forgot
    BWT = sum(R[-1, i] - R[i, i] for i in range(T-1)) / (T-1)
    
    # Forgetting: max accuracy minus final accuracy
    F = np.mean([np.max(R[:, j]) - R[-1, j] for j in range(T)])
    
    return AA, BWT, F
```
